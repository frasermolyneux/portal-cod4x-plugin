# Plugin Runtime

## Local configuration

The runtime loads `portal-cod4x-plugin.config.json` during startup. Required fields:

- `tenantId`
- `clientId`
- `clientSecret`
- `repositoryApiBaseUrl`
- `repositoryApiResource`
- `gameServerId`
- `refreshIntervalSeconds` (15-900)

If the file is missing or invalid, runtime settings retrieval is paused and retried with backoff.

## Startup broadcast behavior

When the plugin is loaded (`OnInit`), it sends a broadcast message to all players using CoD4x chat:

- Chat prefix: `^4[^1XI-BOT^4]^7`
- Broadcast format: `^4[^1XI-BOT^4]^7 Portal Plugin is online (version <version>)`
- Startup log format: `Portal Plugin is online (version <version>)`
- Target slot: `-1` (broadcast/all players)

The startup message is generated by `BuildOnlineBroadcastMessage` and dispatched by `InitializePlugin`.

## Scheduled settings loop

After startup, the runtime performs periodic refreshes:

1. Acquire Entra access token via client-credentials.
2. Fetch global `cod4xCommands` configuration.
3. Fetch server-level `cod4xCommands` override for configured `gameServerId`.
4. Build effective command map: built-in defaults -> global overrides -> server overrides.
5. Reconcile command powers only when effective `cod4xCommands.enabled` is true.

When effective `cod4xCommands.enabled` is false, reconciliation is skipped and existing in-server command powers are left unchanged.

## CoD4x ABI wiring

The plugin exports:

- `OnInfoRequest(pluginInfo_t*)`
- `OnInit()`
- `OnFrame()`
- `OnClientAuthorized()`
- `OnTerminate()`

`OnInfoRequest` advertises handler compatibility `4.000` and plugin `major.minor` derived from build-time version settings.

`OnFrame` and `OnClientAuthorized` both pump runtime `TickPlugin` so periodic settings refresh can continue while the server is active.

## Version source

Version is sourced from `version.json` and injected by CMake into compile definitions:

- `PORTAL_COD4X_PLUGIN_VERSION`
- `PORTAL_COD4X_PLUGIN_VERSION_MAJOR`
- `PORTAL_COD4X_PLUGIN_VERSION_MINOR`

This ensures the online broadcast and plugin metadata remain consistent with the configured plugin build.
